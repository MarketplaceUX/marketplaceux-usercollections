My Account Page

Mobile – Collections Tab

  1. Navigation
    The Collections tab can be accessed by selecting the user’s avatar, then selecting “Apps & Collections” menu item on the navigation bar.
  2. Create collection message
    • If user has no collection, show a message that reads “Create collections of your favourite apps! Share your collections with friends and the Marketplace community.”
    • If user has at least one collection, do not show this message.
  3. Create a new collection button
    • If user has no collection, the button should appear below the message.
    • If user has at least one collection, the button should appear below the Collections/Wishlist/My Apps navigation tabs.
  4. List of collection
    Collections should be sorted from the most recently created to the oldest.
  5. Statistics
    The collection creator should be able to view the stats for their collection. These stats include:
    • Number of shares the collection has received
    • Number of views the collection has received
    • Number of upvotes the collection has received
    6. WARNING: The interface to do this is not yet known. The mechanism to share his/her own collection is also unclear. We need to figure this out.
  6. Search bar
    • Only show search when user has created more than 3 collections.
    • Search should appear above the list of collections and below the Create a new collection button
    • The search box should appear just above the list of collections.

Mobile – Wishlist Tab

Every user has an app wishlist by default.

  1. Navigation
    The Wishlist tab can be accessed by selecting the user’s avatar, then selecting “Apps & Collections” menu item on the navigation bar.
  2. Wishlist message
    • If the user has no app in the wishlist, show a message that reads “Nothing in your wishlist? Add any apps to your wishlist by tapping the ⊕ icon on an app detail page.”
    • If user has at least one app in the wishlist, do not show this message.
  3. Edit list button
    When selected, this button takes user to the wishlist editing interface.
  4. List of wishlisted apps
    • Wishlisted apps should be sorted from the most recently added to the oldest.
    • If an app on the wish list is installed by the user the app should be automatically removed from the user's app wishlist.
  5. Done button
    This button only appears on the wishlist editing interface. When selected, this button takes user back to the wishlist viewing interface.
  6. Remove an app
    • This button only appears on the wishlist editing interface.
    • When selected, an app is removed from the wishlist

My Apps Tab: Mobile

This page lists apps that the user has installed on her/his devices.

  2. My Apps message

    • If the user has more than one app installed, do not show this message.
  3. Remove an app

Collections, Wishlist & My Apps Screen: Desktop

Acceptance criteria for viewing your user collections on desktop

  • Collections are visible under “My Account” when the user is signed in.
  • On desktop the user can see Collections, Wishlist and My Apps in one view.
  • On desktop the sections of the page have titles
  • If there are > 3 collections show a search bar so that the user can search through their collections.
  • Create new collection call-to-action should take user to the create a collection screen.
  • Collections are listed from most recently created to oldest in blocks of 6 moving left to right, up to down.
  • The collection graphic should be the same as seen on mobile.
  • If the user has > 6 collections the collections should start being visible as overflow content. A right arrow control should allow the user to scroll through the overflow content.
  • If the user has 0 collections show text to let the user know what collections are.

Collections, Wishlist & My Apps Screen: Tablet

Acceptance criteria for viewing your user collections on tablet

Same as for desktop, except:

  • Collection search is below Collection title
  • Only 4 collections are visible at a time
